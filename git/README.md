# git命令

```

git add . // 将工作区所有文件内容添加到暂存区
git restore . // 将暂存区内容覆盖到当前
git rm --cache 文件路径 // 删除暂存区某文件内容

git reset // 取消暂存区所有内容（不加后面的版本号）

```

# 文件状态：

1. U 未跟踪
2. A 新添加
3. M 已修改
   4 '' 空，未修改

```
git status -s // 当前文件状态
// 输出： MM xxx 文件 ；
//  第一个M 在暂存区状态 第二个M 工作区状态
```

# git回退版本的3种模式

ps: 如果想要测试，可以先创建一个测试文件A.html,commit一次测试文件，依次执行下面几个命令，就可以看到区别

```
git log --oneline // 查看提交历史 commit

// 当前提交 A->B->C

git reset --soft B版本号 // 想撤销最近几次 历史提交，但保留所有修改并继续改/重新. 改动全在暂存区。

git reset B版本号 //  与 git reset --mixed 等价. 想撤销 历史提交，改动保留在磁盘上，但先不要暂存（重新挑选要 add 的文件）。

git reset --hard B版本号
//  想工作区、暂存区、分支都回到某个干净提交，不要那些未提交的修改。
// 慎用：会丢本地已跟踪文件的修改；未 git add 的新文件默认不会被删。



```
